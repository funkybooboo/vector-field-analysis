\section{Conclusion}

Our comparative analysis reveals that while shared-memory implementations (POSIX Threads and OpenMP) provide incremental performance gains of approximately 6--7\% at low thread counts, they are ultimately constrained by a high serial fraction ($\approx$ 87\%).
The overhead of thread management and the sequential requirements of the final path reconstruction phase limit the scalability of these CPU-based parallel strategies.

In contrast, the massively parallel CUDA implementation achieved the most significant performance improvement, providing a 1.90$\times$ speedup over the sequential reference.
The GPU's ability to handle millions of cells simultaneously through optimized block sizes and device-side atomic synchronization successfully mitigated much of the tracing overhead, although performance remained bound by host-to-device memory transfer latency and the non-parallelizable serialization of output data.

The distributed-memory implementation using OpenMPI highlighted the significant challenges of communication of memory across isolated processes.
The high communication cost associated with iterative synchronization resulted in a speedup of slower than a serial implementation ($0.76\times$).

Crucially, all parallel implementations achieved bitwise parity with the serial reference, demonstrating that  performance acceleration can be achieved without compromising the numerical integrity of the flow analysis.
Ultimately, while GPU acceleration provides the most immediate path to performance gains, the development of truly scalable distributed streamline visualization requires minimizing global synchronization overhead.
